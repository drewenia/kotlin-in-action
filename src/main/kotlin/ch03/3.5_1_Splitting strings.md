# Splitting strings

Muhtemelen String üzerindeki split method’una aşinasındır. Herkes onu kullanır, ancak bazen insanlar Stack Overflow’da
bundan şikâyet eder: “Java’daki split method’u nokta ile çalışmıyor.” Yaygın bir tuzak, "12.345-6.A".split(".") yazıp
sonuç olarak [12, 345-6, A] array’ini beklemektir. Ancak Java’nın split method’u boş bir array döndürür! Bunun nedeni,
parameter olarak bir regular expression alması ve string’i bu expression’a göre birden fazla string’e bölmesidir. Burada
nokta (.), herhangi bir karakteri ifade eden bir regular expression’dır. Kotlin, kafa karıştırıcı olan bu method’u
gizler ve yerine farklı argument’ler alan, split adında birden fazla overloaded extension sağlar. Regular expression
alan olanı, String değil Regex ya da Pattern type’ında bir value ister. Bu da bir method’a geçirilen string’in plain
text mi yoksa regular expression olarak mı yorumlandığının her zaman net olmasını sağlar. İşte string’i ya nokta ya da
tire ile nasıl split edeceğin:

```kotlin
fun main() {
    val text = "12.345-6.A"
    val splittedText = text.split("\\.|-".toRegex())
    println(splittedText) // [12, 345, 6, A]
}
```

Kotlin, Java ile **tam olarak aynı** regular expression syntax’ını kullanır. Buradaki pattern, bir nokta (literal bir
karakteri kastettiğimizi belirtmek için escape ettik, wildcard değil) ya da tire ile eşleşir. Regular expression’larla
çalışmaya yönelik API’ler de standart Java library API’lerine benzer, ancak daha idiomatiktir. Örneğin Kotlin’de, bir
string’i regular expression’a dönüştürmek için toRegex adlı bir extension function kullanırsın. Ancak bu kadar basit bir
durum için regular expression kullanmana gerek yoktur. Kotlin’de split extension function’ının bir diğer overload’u,
plain-text string’ler olarak arbitrary sayıda delimiter alır:

```kotlin
fun main() {
    val text = "12.345-6.A"
    val splittedText = text.split(".","-")
    println(splittedText) // [12, 345, 6, A]
}
```

Character argument’ler de belirtebileceğini ve "12.345-6.A".split('.', '-') yazabileceğini unutma; bu da aynı sonucu
verir. Bu method, yalnızca tek bir character’ı delimiter olarak alabilen benzer Java method’unun yerini alır.
