# Named arguments

Ele alacağımız ilk problem, function call’larının readability’siyle ilgilidir. Örneğin, aşağıdaki joinToString
call’una bak: ```joinToString(collection, " ", " ", ".")```

Bu String’lerin hangi parameter’lara karşılık geldiğini söyleyebilir misin? Element’ler whitespace ile mi yoksa nokta
ile mi ayrılıyor? Bu sorular, function’ın signature’ına bakmadan cevaplaması zor sorulardır. Belki hatırlıyorsundur
ya da IDE’n yardımcı olabilir, ama calling code’dan bu açık değildir. Bu problem özellikle Boolean flag’lerle sıkça
görülür. Bunu çözmek için bazı Java coding style’ları, Boolean kullanmak yerine enum type’lar oluşturmayı önerir.
Diğerleri ise aşağıdaki example’da olduğu gibi parameter name’lerini bir comment içinde açıkça belirtmeni bile ister.

```
joinToString(collection, /* separator */ " ", /* prefix */ " ", /* postfix */ ".");
```

Kotlin ile daha iyisini yapabilirsin: ```joinToString(collection, separator = " ", prefix = " ", postfix = ".")```

Kotlin’de yazılmış bir function’ı call ederken, function’a geçirdiğin bazı (veya tüm) argument’lerin isimlerini
belirtebilirsin. Eğer function’a geçen tüm argument’lerin isimlerini belirtirsen, hatta onların sırasını bile
değiştirebilirsin:

```kotlin
joinToString(
    postfix = ".",
    separator = " ",
    collection = collection,
    prefix = " "
)
```

Named argument’lar, özellikle default parameter value’larla çok iyi çalışır; buna bir sonraki adımda bakacağız.
