# Return statements in lambdas: returning from an enclosing function

Bir collection üzerinde iterate etmenin iki farklı yolunu karşılaştıracağız. Aşağıdaki code’da, eğer person’ın adı
Alice ise, lookForAlice function’ından return ettiğiniz açıktır.

```kotlin
data class Person(val name: String, val age: Int)

val people = listOf(
    Person("Alice", 29),
    Person("Bob", 31)
)

fun lookForAlice(people: List<Person>) {
    for (p in people) {
        if (p.name == "Alice") {
            println("Found!")
            return
        }
    }
    println("Alice is not found")
}

fun main() {
    lookForAlice(people) // Found!
}
```

Bu code’u forEach iteration kullanarak yeniden yazmak güvenli mi? return statement aynı anlama mı gelir? Evet, forEach
function’ını kullanmak güvenlidir, aşağıda gösterildiği gibi.

```kotlin
data class Person(val name: String, val age: Int)

val people = listOf(
    Person("Alice", 29),
    Person("Bob", 31)
)

fun lookForAlice(people: List<Person>) {
    people.forEach {
        if (it.name == "Alice") {
            println("Found!")
            // Diğer code’da olduğu gibi, function’dan return eder.
            return
        }
    }
    println("Alice is not found")
}

fun main() {
    lookForAlice(people) // Found!
}
```

Lambda içinde return keyword’ünü kullanırsanız, yalnızca lambda’dan değil, lambda’yı çağırdığınız function’dan return
eder. Bu tür return statement’a non-local return denir; çünkü return statement’ını içeren bloktan daha büyük bir bloktan
return eder.

Kuralın mantığını anlamak için, bir Java method’unda for loop veya synchronized block içinde return keyword’ü kullanmayı
düşünün. Return’un loop veya block’tan değil, function’dan döndüğü açıktır. Kotlin, dil özelliklerinden lambda argument
alan function’lara geçtiğinizde aynı davranışı korumanıza olanak tanır.

Outer function’dan return edilmesi yalnızca, lambda’yı argument olarak alan function inline ise mümkündür. forEach
function’ının body’si lambda’nın body’siyle birlikte inline edilir; bu sayede return expression’ı, enclosing
function’dan return edecek şekilde kolayca compile edilebilir. Lambda’lar non-inline function’lara geçirildiğinde return
expression kullanmak izinli değildir; çünkü non-inline function, kendisine geçirilen lambda’yı bir variable’da
saklayabilir. Bu durumda, function zaten return ettikten sonra lambda çalıştırılabilir, dolayısıyla lambda’nın
surrounding function’ın return zamanını etkilemesi mümkün olmaz.
