# Variance describes the subtyping relationship between generic arguments

Variance kavramı, aynı base type’a sahip ve farklı type argument’ları olan type’ların birbirleriyle nasıl ilişkili
olduğunu açıklar: örneğin, List<String> ve List<Any>. Öncelikle, bu ilişkinin genel olarak neden önemli olduğunu
tartışacağız, ardından Kotlin’de nasıl ifade edildiğine bakacağız. Variance’ı anlamak, kendi generic class veya
function’larını yazarken çok önemlidir: bu, kullanıcıları rahatsız edici şekillerde sınırlamayan ve type-safety
beklentilerini bozmayan API’ler oluşturmanıza yardımcı olur.
