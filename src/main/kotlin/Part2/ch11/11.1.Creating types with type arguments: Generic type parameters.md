# Creating types with type arguments: Generic type parameters

Generic'ler, type parameter’lara sahip type’lar tanımlamana olanak tanır. Böyle bir type’ın bir instance’ı
oluşturulduğunda, type parameter’lar type argument olarak adlandırılan belirli type’larla değiştirilir. Örneğin, List
type’ında bir variable’ın varsa, bu list’in içinde ne tür şeylerin saklandığını bilmek faydalıdır. type parameter tam
olarak bunu belirtmeni sağlar—“Bu variable bir list tutuyor” demek yerine, “Bu variable string’lerden oluşan bir list
tutuyor” gibi bir ifade kullanabilirsin. Kotlin’de “string’lerden oluşan bir list” demenin sözdizimi Java’dakiyle
aynıdır: List<String>. Bir class için birden fazla type parameter da declare edebilirsin. Örneğin, Map class’ı key type
ve value type için type parameter’lara sahiptir: class Map<K, V>. Bunu belirli argument’larla instantiate edebiliriz:
Map<String, Person>. Şu ana kadar her şey Java’dakiyle tamamen aynıdır.

Genel olarak type’larda olduğu gibi, type argument’lar da çoğu zaman Kotlin Compiler tarafından infer edilebilir:

```kotlin
val authors = listOf("Sveta", "Seb", "Dima", "Roman")
```

listOf fonksiyonuna geçirilen tüm value’lar string olduğu için, Compiler bir List<String> oluşturduğunu infer eder:

![img.png](assets/img.png)

Öte yandan, empty bir list oluşturman gerektiğinde, type argument’ı infer edecek hiçbir şey yoktur; bu yüzden onu
explicitly belirtmen gerekir. Bir list oluşturma durumunda, type’ı variable declaration’ın bir parçası olarak belirtmek
ile list oluşturan fonksiyon için bir type argument belirtmek arasında bir seçeneğin vardır. Aşağıdaki example, bunun
nasıl yapıldığını gösterir:

```kotlin
val readers: MutableList<String> = mutableListOf()
val readers = mutableListOf<String>()
```

Bu declaration’lar eşdeğerdir.

### There are no raw types in Kotlin

Java’nın aksine, Kotlin type argument’ların ya explicitly belirtilmesini ya da Compiler tarafından infer edilmesini her
zaman zorunlu kılar. Generics Java’ya yalnızca 1.5 sürümünde eklendiği için, daha eski sürümler için yazılmış code’larla
uyumluluğu korumak zorundaydı; bu nedenle type argument olmadan bir generic type kullanmana izin verir—raw type olarak
adlandırılan şey. Örneğin, Java’da içinde ne tür şeyler barındırdığını belirtmeden ArrayList type’ında bir variable
declare edebilirsin:

```java
List aList = new ArrayList();
```

Kotlin, en başından beri generics’e sahip olduğu için raw type’ları desteklemez ve type argument’ların her zaman
tanımlanmış olması gerekir. Programın Java code’undan raw type’a sahip bir variable alırsa, bu variable, type’ı Any!
olan bir generic parameter’a sahipmiş gibi ele alınır — section 7.12’de tanıdığın üzere bir platform type olarak.
