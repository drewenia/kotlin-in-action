# Declaring accessors with reified type parameters

Function'lar, Kotlin’de inline olabilen ve reified type parameters kullanabilen tek construct’lar değildir. 2.2.2
bölümünde property accessors’ın getter ve setter’lar için custom implementasyonlar sağlayabildiğini zaten gördün. Eğer
bir property accessor generic type üzerinde define edilmişse, property’yi inline ve type parameter’ı reified olarak
işaretlemek, type argument olarak kullanılan spesifik class’a referans vermene olanak tanır.

Bu örnekte, generic bir class’ın canonical name’ini döndüren canonical adlı bir extension property sağlıyorsun. 11.2.3
bölümünde olduğu gibi, bu, T::class.java çağrısını sarmalayarak canonicalName property’sine erişmek için daha kullanışlı
bir yol sunar.

```kotlin
inline val <reified T> T.canonical: String
    get() = T::class.java.canonicalName

fun main() {
    println(listOf(1, 2, 3).canonical) // java.util.List
    println(123.canonical) // java.lang.Integer
}
```
