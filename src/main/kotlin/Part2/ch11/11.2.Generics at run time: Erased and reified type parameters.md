# Generics at run time: Erased and reified type parameters

Implementation açısından, Java virtual machine (JVM) üzerindeki generics genellikle type erasure ile implement edilir.
Bu, generic bir class’ın bir instance’ının type argument’larının runtime’da korunmadığı anlamına gelir. Bu section’da,
Kotlin için type erasure’in pratik etkilerini ve inline bir fonksiyon declare ederek sınırlamalarının nasıl
aşılabileceğini ele alacağız. Inline bir fonksiyon declare ederek, type argument’larının erase edilmemesini (Kotlin
terimiyle, reified olmasını) sağlayabilirsin. Reified type parameter’ları detaylı olarak ele alacak ve kullanışlı
oldukları durumlara örnekler göreceğiz.
