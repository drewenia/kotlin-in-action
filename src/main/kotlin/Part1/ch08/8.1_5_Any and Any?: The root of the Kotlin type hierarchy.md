# Any and Any?: The root of the Kotlin type hierarchy

Java’da Object class hiyerarşisinin kökü olduğu gibi, Any type da Kotlin’deki tüm non-nullable type’ların
supertype’ıdır. Ancak Java’da Object yalnızca tüm reference type’ların supertype’ıdır ve primitive type’lar hiyerarşinin
bir parçası değildir. Bu, Object gerektiğinde primitive type value’yu represent etmek için java.lang.Integer gibi
wrapper type’ları kullanman gerektiği anlamına gelir. Kotlin’de ise Any, Int gibi primitive type’lar dahil tüm
type’ların supertype’ıdır. Java’da olduğu gibi, bir primitive type’ın value’sını Any type’ına sahip bir variable’a
assign etmek otomatik boxing gerçekleştirir:

```kotlin
// 42 value’su boxed edilir; çünkü Any bir reference type’tır.
val answer: Any = 42
```

Any bir non-nullable type’tır; bu nedenle Any type’ına sahip bir variable null value tutamaz. Kotlin’de her olası
value’yu, null dahil, tutabilen bir variable’a ihtiyaç duyarsan Any? type’ını kullanman gerekir.

Perde arkasında, Any type java.lang.Object’a karşılık gelir. Java method’larının parameter ve return type’larında
kullanılan Object type, Kotlin’de Any olarak görülür. (Daha spesifik olarak, nullability’si bilinmediği için platform
type olarak değerlendirilir.) Bir Kotlin function Any kullanıyorsa, Java bytecode’unda Object olarak compiled edilir.

Bildiğin gibi, tüm Kotlin class’larının aşağıdaki üç method’u vardır: toString, equals ve hashCode. Bu method’lar
Any’den inherit edilir. java.lang.Object üzerinde defined olan diğer method’lar (örneğin wait ve notify) Any üzerinde
mevcut değildir, ancak value’yu manuel olarak java.lang.Object’a cast edersen çağırabilirsin.
