# The Nothing type: “This function never returns”

Kotlin’de bazı function’lar için “return value” kavramı anlamlı değildir; çünkü bu function’lar asla başarılı şekilde
tamamlanmaz. Örneğin, birçok testing library’de fail adında bir function vardır; bu function, belirtilen bir message ile
exception fırlatarak mevcut testi başarısız kılar. İçinde sonsuz loop olan bir function da asla başarılı şekilde
tamamlanmaz.

Böyle bir function’ı çağıran kodu analiz ederken, function’ın normal şekilde asla tamamlanmayacağını bilmek faydalıdır.
Bunu ifade etmek için Kotlin, Nothing adında özel bir return type kullanır:

```kotlin
fun fail(message: String): Nothing {
    throw IllegalArgumentException(message)
}

fun main() {
    fail("Error occured")
    // java.lang.IllegalStateException: Error occurred
}
```

Nothing type’ın hiç value’su yoktur; bu nedenle yalnızca bir function’ın return type’ı olarak veya generic function
return type olarak kullanılan type parameter için type argument olarak kullanılması anlamlıdır. Diğer tüm durumlarda,
herhangi bir value saklayamayacağın bir variable declare etmek anlamlı değildir.

Nothing döndüren function’lar, precondition kontrolü yapmak için Elvis operator’ün sağ tarafında kullanılabilir:

```kotlin
val address = company.address ?: fail("No address")
println(address.city)
```

Bu örnek, type sisteminde Nothing’in bulunmasının neden son derece faydalı olduğunu gösterir. Compiler, return type’ı
Nothing olan bir function’ın normal şekilde tamamlanmayacağını bilir ve function’ı çağıran kodu analiz ederken bu
bilgiyi kullanır. Önceki örnekte, compiler address’in type’ının non-null olduğunu infer eder; çünkü null olduğu durumu
handle eden branch her zaman exception fırlatır ve takip eden kodun çalışmasını devam ettirmez. Kotlin standard
library’de de böyle bir function vardır: error function IllegalStateException fırlatır ve return type’ı Nothing’tir.

Kotlin’deki basic type’lar hakkındaki tartışmamızı tamamladık: primitive type’lar, Any, Unit ve Nothing. Şimdi,
collection type’larına ve bunların Java’daki karşılıklarından nasıl farklı olduklarına bakalım.
